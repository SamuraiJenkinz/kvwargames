<objective>
Install React Router v7, scaffold the routed app shell (`/setup`, `/game`, catch-all), add placeholder SetupScreen and GameScreen components, and remove the now-redundant `phase` field from the Zustand store. This unblocks plans 04-02, 04-03, and 04-04 (all of which assume the router and the placeholder screens exist) and removes the navigation-state duplication identified in research.

Purpose: The current app has no router — `main.tsx` renders `<App />` directly and the only way to "navigate" is the store's `phase` field. Phase 4 makes URL the canonical navigation source, and every subsequent UI plan in this phase mounts components inside `/setup` or `/game`. This plan establishes that foundation in one atomic step.

Output: A booting router-driven app, a clean store with no `phase` duplication, and two placeholder route components ready to be filled in by 04-02/04-03 and Phase 5.
</objective>

   - `/setup` → renders `<SetupScreen />`
   - `/game` → renders a guarded element (see Task below): use a tiny inline component (or inline guard expression in the route element) that reads `useGameStore((s) => s.gameState)` and returns `<Navigate to="/setup" replace />` when null, otherwise `<GameScreen />`. The `replace` prop is critical — without it, the redirect creates an infinite back-button loop.
   - `*` (catch-all) → `<Navigate to="/setup" replace />`
   - `/` → `<Navigate to="/setup" replace />` (explicit, even though the catch-all would also handle it)
   Imports: `Routes`, `Route`, `Navigate` from `"react-router"`.

4. Create `src/components/setup/SetupScreen.tsx` — placeholder export. For now render a minimal centred container with the app title (e.g. "EDIP Wargame Facilitator") and the text "Setup screen — content arrives in plan 04-02". This is replaced in 04-02. Use existing Tailwind tokens (`bg-bg-base text-text-primary`) so it doesn't look broken.

5. Create `src/components/game/GameScreen.tsx` — placeholder export. Render a minimal page that displays "Game screen — three-column layout arrives in Phase 5" plus a small dev-only readout of `useGameStore((s) => s.gameState?.scenarioIndex)` so the route is observably wired to the store. No game logic, no chrome.

Do NOT touch `TokenReference.tsx` — it remains a dev-only component. It should no longer be the App's render target after this plan.

The store currently duplicates navigation state via `phase: AppPhase`. Now that React Router owns navigation (Task 1), this field is redundant and must be removed to prevent two-source-of-truth bugs. RESEARCH.md confirms this is safe: only the store and its tests reference `phase`.

1. In `src/lib/gameStore.ts`:
   - Remove the `phase: AppPhase` field, the `setPhase` method, and the `phase` import from the type import line.
   - Remove `state.phase = 'game'` from inside `initGame`.
   - Remove `state.phase = 'setup'` from inside `resetGame`.
   - Keep `setupMode` and `setSetupMode` exactly as-is — they have no URL equivalent in Phase 4 (the home/load/brief/review distinction is intra-`/setup`).
   - Keep the `AppPhase` type export in `src/types/game.ts` for now — it's harmless and may be useful in Phase 5 for `'game' | 'debrief'` distinction within `/game`. (If you'd rather delete it, that's fine too — grep first to confirm nothing imports it. As of this plan, only `gameStore.ts` did.)

2. In `src/lib/gameStore.test.ts`:
   - Delete the test `'phase is setup'` (lines around 24–26).
   - Delete the test `'sets phase to game'` (lines around 69–71).
   - Delete the test `'returns phase to setup'` (lines around 163–165).
   - Delete the entire `setPhase changes phase` test (lines around 368–371). If its surrounding `describe` block becomes empty, delete the describe too.
   - All other tests stay — `setupMode`, `gameState`, `gameConfig`, `initGame` results, `resetGame` results, etc. remain the meaningful contract.

3. Open `src/components/game/GameScreen.tsx` (created in Task 1) — confirm it does not reference `phase`. If you wired any temporary `phase` read, remove it.

4. Search the entire repo to confirm no other references:
