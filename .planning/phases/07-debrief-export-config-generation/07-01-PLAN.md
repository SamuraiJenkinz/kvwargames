<decisions_locked>
1. **`## Debrief` section anchor** = LAST `debrief_divider` in messages (NOT first). Research pitfall 7 proves the first-divider anchor wrongly swallows post-interim round content. Code this as `messages.lastIndexOf(msg => msg.type === 'debrief_divider')`. Document the choice in a JSDoc comment on `generateDebriefMarkdown`.
2. **`gameEnded` lives on `GameStore`, not `GameState`** — session UI state, not simulation state. Research pitfall 6. Reset to `false` in `newGame()` AND `resetGame()`. **NOTE — CONTEXT.md override:** CONTEXT.md's "Debrief trigger UX" section (~line 47) incorrectly lists `gameEnded: boolean` on `GameState`. This plan overrides CONTEXT.md on that one point: `gameEnded` is session UI state and lives on `GameStore`. Research pitfall 6 explicitly discusses this trap. If an executor sees the CONTEXT.md placement, ignore it and follow this plan.
3. **Snapshot capture uses `immer.current()`** imported from `'immer'` inside `advanceRound`'s `set()` callback — `structuredClone` on an Immer draft Proxy throws `DataCloneError`. Research pitfall 1.
4. **Snapshot keying convention (Option A — "keyed by the round being ENTERED")**: `stateSnapshots[N]` always represents "state at the START of Round N". `initGame` seeds `stateSnapshots[1]` with `current(state.gameState)` after field assignments. `advanceRound` captures `stateSnapshots[newRound] = current(s.gameState)` AFTER it bumps `s.gameState.round = newRound` and pushes the round_divider (i.e. the snapshot captures the exact state that round N+1 begins with). The exporter's `renderRound(n, ...)` reads `stateSnapshots[n]` directly with no off-by-one adjustment. This convention is used consistently across both plans 07-01 and 07-02.
5. **`downloadDebrief` defers `URL.revokeObjectURL`** with `setTimeout(() => URL.revokeObjectURL(url), 0)` — Firefox blob-read race. Research pitfall 2. Tests use `vi.useFakeTimers()` + `vi.runAllTimers()` to deterministically flush the revoke (no microtask-flush fallback — lock to fake timers to avoid flakiness).
6. **Zero new npm deps** — Blob + URL.createObjectURL + synthetic anchor pattern. Research "Don't Hand-Roll" table.
7. **Per-round snapshot table format** = GFM pipe table (`| Team | PC | PO | RDY | STK | CRM | IC |`) — readable in VS Code preview AND GitHub AND plain text. Research code example.
8. **Filename format** = `debrief-{kebab-game-name}-{YYYY-MM-DD-HHmm}.md` via `buildDebriefFilename(gameName, date)`.
9. **Appendix** — include a one-line `## Appendix: Raw Config` section at the bottom: game name + scenario index + scenario name. "Lean toward yes" per CONTEXT.md Claude's Discretion.

         const blob = new Blob([markdown], { type: 'text/markdown; charset=utf-8' })
