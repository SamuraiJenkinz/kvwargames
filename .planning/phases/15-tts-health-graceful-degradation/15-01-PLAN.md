**Step 2 ‚Äî Create `backend/app/routers/health_tts.py`:**

Use `backend/app/routers/health.py` as the structural template. Copy the module-docstring shape, the handler-order pattern (TimeoutException ‚Üí HTTPStatusError ‚Üí ConnectError ‚Üí RequestError ‚Üí Exception), the shared-failure-return-path pattern (lines 193-211 of health.py), and the `JSONResponse(status_code=200, ...)` invariant.

**Critical differences from `health.py`:**

1. **DO NOT reuse `request.app.state.http_client`** ‚Äî that is the LLM client with LLM-specific defaults. Create a fresh `httpx.AsyncClient` per-request via `async with httpx.AsyncClient(timeout=httpx.Timeout(15.0)) as client:` context manager. Reason: RESEARCH.md Pitfall 1 ‚Äî coupling LLM and TTS transports is a maintainability trap. Per-request client is acceptable because the 30s cache keeps call frequency low.

2. **Probe URL is `https://api.elevenlabs.io/v1/user`** (HARDCODE this constant at module level ‚Äî do not pull from settings). The v1/user endpoint is smaller than /v1/voices (profile JSON vs. ~50KB voice list) while exercising the same auth surface. RESEARCH.md Q4 documents `/v1/voices` as the fallback if /v1/user proves unreliable in the field.

3. **Auth header is `xi-api-key: {settings.elevenlabs_api_key}`** ‚Äî confirmed from ElevenLabs SDK source (`client_wrapper.py`). Do NOT use `Authorization: Bearer`.

4. **TTS_PROVIDER=fake short-circuit (LOCKED DECISION from orchestrator inputs #6 + RESEARCH.md Pitfall 3):** At the top of the handler, BEFORE reading the cache or firing any network call, check `if settings.tts_provider == "fake":` and return `JSONResponse(status_code=200, content={"ok": True, "latencyMs": 0})` immediately. This avoids constant amber false-alarm on dev setup screens. Log at INFO level: `logger.info("tts_health_check code=ok status=None latencyMs=0 provider=fake")`.

5. **30-second in-memory cache (module-level tuple + asyncio.Lock):**

```python
import asyncio
import time

_CACHE_TTL_S = 30.0
_cache_lock = asyncio.Lock()
_cache: "tuple[float, dict] | None" = None  # (monotonic_timestamp, response_body_dict)
```

Cache read: inside `async with _cache_lock:`, check `_cache is not None and (time.monotonic() - _cache[0]) < _CACHE_TTL_S`. If hit, return the cached body immediately via `JSONResponse(status_code=200, content=cached_body)`.

Cache write: after any probe completes (success OR failure), update `_cache = (time.monotonic(), response_body_dict)` inside the lock.

6. **`?force=true` query param bypasses cache READ only, still WRITES on completion** (RESEARCH.md Pitfall 7). Signature: `async def tts_health_check(force: bool = False) -> JSONResponse`. When `force=True`, skip the cache-read branch but always write the new result back to cache.

7. **Error-code mapping (mirror health.py lines 132-191 exactly, swap hints to reference ELEVENLABS_API_KEY and api.elevenlabs.io):**
   - `httpx.TimeoutException` ‚Üí `code="timeout"`, hint="ElevenLabs did not respond within 15 seconds ‚Äî check network or provider status"
   - `httpx.HTTPStatusError` 401/403 ‚Üí `code="auth_error"`, hint="Authentication failed ‚Äî check ELEVENLABS_API_KEY in .env"
   - `httpx.HTTPStatusError` 404 ‚Üí `code="not_found"`, hint="ElevenLabs endpoint not found ‚Äî probe URL may have changed"
   - `httpx.HTTPStatusError` 429 ‚Üí `code="rate_limited"`, hint="Rate limited by ElevenLabs ‚Äî retry in a moment"
   - `httpx.HTTPStatusError` other ‚Üí `code="upstream_error"`, hint=f"ElevenLabs provider error (HTTP {status}) ‚Äî try again or check provider status"
   - `httpx.ConnectError` with `ssl.SSLError` `__cause__` ‚Üí `code="tls_error"`, hint="TLS handshake failed ‚Äî check corporate proxy and custom CA settings"
   - `httpx.ConnectError` other ‚Üí `code="network_error"`, hint="Cannot reach api.elevenlabs.io ‚Äî check network connectivity and firewall"
   - `httpx.RequestError` (catch-all) ‚Üí `code="network_error"`, hint="Cannot reach api.elevenlabs.io ‚Äî check network connectivity and firewall"
   - `Exception` (unexpected body shape, e.g. missing `subscription` key) ‚Üí `code="invalid_response"`, status=200, hint="ElevenLabs returned an unexpected response shape"

8. **Response validation:** After `response.raise_for_status()`, call `data = response.json()` and assert `"subscription" in data`. If the key is missing, raise `ValueError("missing subscription key")` ‚Äî this falls through to the `Exception` handler ‚Üí `invalid_response`.

9. **Logging:** Every success and every failure branch logs via `logger.info("tts_health_check code=%s status=%s latencyMs=%d", code, status, latency_ms)` ‚Äî same shape as health.py.
