New behaviour — preserve production contract, add DEV-only shortcut:

```tsx
import { seedMockState } from '@/mocks/seedMockState'

function GuardedGameScreen() {
  const gameState = useGameStore((s) => s.gameState)

  if (gameState === null) {
    if (import.meta.env.DEV) {
      // Dev convenience: seed mock state synchronously so the game screen
      // renders immediately without going through the Launch flow.
      seedMockState()
      // Fall through and render below — the next render will have state.
      return null
    }
    return <Navigate to="/setup" replace />
  }

  return <GameScreen />
}
```

Note: Calling `seedMockState()` during render is safe here because:
- It only runs when `gameState === null` (once per navigation, not continuously)
- Returning `null` immediately after seed causes an immediate re-render with seeded state
- In production the branch is stripped by `import.meta.env.DEV` dead-code elimination

Add a test (or extend existing App.test.tsx if present) that mocks `import.meta.env.DEV = true` scenario is covered by the existing dev environment of Vitest — no extra test needed here; the critical production invariant (null gameState redirects to /setup in non-dev builds) is already covered by Phase 4's 04-04 tests. If STATE.md mentions those tests, leave them alone; verify they still pass.

Do NOT add a useEffect or setTimeout — the render-phase seed + `return null` pattern is clean because the store update triggers a synchronous re-render via Zustand subscription.
