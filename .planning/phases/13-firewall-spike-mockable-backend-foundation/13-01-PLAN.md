**Step 1 — Create the spike script**

Create `.planning/phases/13-firewall-spike-mockable-backend-foundation/scripts/run_firewall_spike.py`. Requirements:

- **No hardcoded secrets.** Read `ELEVENLABS_API_KEY` and `ELEVENLABS_VOICE_KENT` from `os.environ`; exit with a clear error (code 2) if either is missing.
- **Use `requests`, not `httpx`.** The point is corporate-firewall TLS posture parity — `requests` is the minimum-dependency tool an operator can run without our app installed. (`httpx` and `requests` share urllib3-adjacent TLS behavior; both honor `HTTPS_PROXY` env vars. For the spike, we want `requests`.)
- **Endpoint:** `https://api.elevenlabs.io/v1/text-to-speech/{voice_id}?output_format=mp3_44100_128`
- **Headers:** `{"xi-api-key": <key>, "Content-Type": "application/json"}` (ElevenLabs uses `xi-api-key`, NOT `Authorization: Bearer`)
- **Payload:** a JSON body with `text` (~500 chars of EDIP-domain prose — example text below) and `model_id: "eleven_multilingual_v2"`. The text should be long enough that the resulting MP3 exceeds 60 seconds at typical English reading rate (~150 wpm); ~500 chars of dense prose comfortably clears this.

Example text to put in the payload (use this verbatim so the output is deterministic across re-runs):

The crisis severity reached level four. Kent Valentina advocated accepting broader EDIP powers. Doctor Alistair Finch flagged the transition to a security related supply crisis. Doctor Michael Chen catalogued the industrial and legal tradeoffs.

- **Timeouts:** Use a generous `timeout=180` seconds on the `requests.post(...)` call — the payload is long enough that short timeouts will false-negative.
- **Write the raw response bytes** to `.planning/phases/13-firewall-spike-mockable-backend-foundation/13-firewall-spike-payload.mp3` using `Path(__file__).parent.parent / "13-firewall-spike-payload.mp3"` relative resolution, opening in binary write mode.

  1. `HTTP status: <status_code>`
  2. `Content-Type: <response.headers.get("Content-Type")>`
  3. `Content-Length (reported): <response.headers.get("Content-Length") or "not set">`
