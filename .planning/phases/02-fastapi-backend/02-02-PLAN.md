  <action>
    Replace the empty router stub in `backend/app/routers/llm.py` with the full LLM proxy endpoint.

    1. Define Pydantic request models:
       - `class Message(BaseModel)`: `role: str`, `content: str`
       - `class LLMProxyRequest(BaseModel)`: `systemPrompt: str`, `messages: list[Message]`, `maxTokens: Optional[int] = None`

    2. Define the endpoint `@router.post("/api/llm")`:
       - Parameters: `body: LLMProxyRequest`, `request: Request`
       - Get settings via `get_settings()`
       - Get httpx client via `request.app.state.http_client`
       - Resolve max_tokens: use `body.maxTokens` if provided, else `settings.llm_max_tokens`

    3. Build the upstream OpenAI-compatible payload:
       ```python
       payload = {
           "model": settings.llm_model,
           "messages": [
               {"role": "system", "content": body.systemPrompt},
               *[m.model_dump() for m in body.messages],
           ],
           "max_tokens": max_tokens,
       }
       ```

    4. Build headers with credential injection:
       ```python
       headers = {
           "Authorization": f"Bearer {settings.llm_api_key}",
           "Content-Type": "application/json",
           **settings.get_extra_headers(),
       }
       ```
       CRITICAL: The API key comes from env, NOT from the request body. This is the entire security model.

    5. Make the httpx POST request inside a try/except chain:
       - `response = await client.post(settings.llm_endpoint_url, json=payload, headers=headers)`
       - `response.raise_for_status()`
       - Extract text: `data = response.json()`, `text = data["choices"][0]["message"]["content"]`
       - Return `{"text": text}`

    6. Exception handling (order matters):
       - `httpx.TimeoutException` -> 504 `{"error": {"code": "LLM_TIMEOUT", "message": f"LLM request timed out after {settings.llm_timeout_seconds}s"}}`
       - `httpx.HTTPStatusError` with status 401 -> 401 `{"error": {"code": "LLM_AUTH_ERROR", "message": "API key missing or rejected by upstream LLM"}}`
       - `httpx.HTTPStatusError` (any other status) -> 502 `{"error": {"code": "LLM_UPSTREAM_ERROR", "message": f"Upstream LLM returned {exc.response.status_code}"}}`
       - `httpx.RequestError` (network/connect failures) -> 502 `{"error": {"code": "LLM_UNREACHABLE", "message": "Could not reach LLM endpoint"}}`
       - `Exception` (catch-all) -> 500 `{"error": {"code": "INTERNAL_ERROR", "message": "Unexpected server error"}}`

    All error responses use `JSONResponse(status_code=..., content=...)` for the consistent error shape.

    AVOID: Do NOT forward raw upstream error bodies — extract status code only.
    AVOID: Do NOT catch `httpx.ReadTimeout` or other subtypes individually — catch the parent `httpx.TimeoutException`.
    AVOID: Do NOT log the API key or include it in error messages.
  </action>
