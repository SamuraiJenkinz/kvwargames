  <action>
    Replace the empty router stub in `backend/app/routers/config_gen.py` with the config generation endpoint.

    1. Define Pydantic request model:
       - `class ConfigGenRequest(BaseModel)`: `brief: str`

    2. Define a `CONFIG_GEN_SYSTEM_PROMPT` constant string that instructs the LLM to:
       - Generate a valid JSON game configuration based on the user's text brief
       - Follow the GameConfig structure (teams, scenarios, cards, national actions, etc.)
       - Return ONLY the JSON — no markdown fences, no explanation
       Keep the prompt concise (~200 words). The exact wording is at Claude's discretion since this will be refined in Phase 6.

    3. Define the endpoint `@router.post("/api/generate-config")`:
       - Parameters: `body: ConfigGenRequest`, `request: Request`
       - Get settings via `get_settings()`
       - Get httpx client via `request.app.state.http_client`

    4. Build the upstream OpenAI-compatible payload:
       ```python
       payload = {
           "model": settings.llm_model,
           "messages": [
               {"role": "system", "content": CONFIG_GEN_SYSTEM_PROMPT},
               {"role": "user", "content": body.brief},
           ],
           "max_tokens": settings.llm_max_tokens,
       }
       ```

    5. Build headers (identical pattern to llm.py):
       ```python
       headers = {
           "Authorization": f"Bearer {settings.llm_api_key}",
           "Content-Type": "application/json",
           **settings.get_extra_headers(),
       }
       ```

    6. Make the httpx POST request with the SAME exception handling chain as llm.py:
       - `httpx.TimeoutException` -> 504 LLM_TIMEOUT
       - `httpx.HTTPStatusError` (401) -> 401 LLM_AUTH_ERROR
       - `httpx.HTTPStatusError` (other) -> 502 LLM_UPSTREAM_ERROR
       - `httpx.RequestError` -> 502 LLM_UNREACHABLE
       - `Exception` -> 500 INTERNAL_ERROR

    7. Extract text from response: `data["choices"][0]["message"]["content"]`
    8. Return `{"text": text}` — raw text, NO JSON parsing server-side

    AVOID: Do NOT parse or validate the LLM's JSON output on the server — that's the frontend's job.
    AVOID: Do NOT share code with llm.py via a helper function yet — keep the duplication explicit. Refactoring to a shared helper is a future improvement, not a Phase 2 concern.
  </action>
