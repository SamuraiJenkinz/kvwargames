**Page structure** (use Tailwind classes, never hardcode hex values):

2. **Persona Colours section**: Three horizontal swatches, each showing:
   - A 48x48 filled square with the persona colour (bg-persona-kent, bg-persona-finch, bg-persona-chen)
   - Persona name label in the persona colour (text-persona-kent, etc.)
   - A sample "tinted panel" showing the recommended treatment: bg-persona-*/8 background with border-l-2 border-persona-*/50

3. **Card Category Colours section**: Seven colour chips in a horizontal row:
   - Each chip is a small rounded-sm rectangle with category colour background
   - Label below each: Crisis State, Monitoring, Prio Soft, Prio Hard, Demand Coord, Production Accel, Transfers
   - Classes: bg-category-crisis, bg-category-monitoring, bg-category-prio-soft, bg-category-prio-hard, bg-category-demand, bg-category-production, bg-category-transfers

4. **Crisis State Badges section**: Three inline badges:
   - "NO CRISIS" badge: bg-crisis-none/20 text-crisis-none border border-crisis-none/30 rounded-sm px-2 py-0.5 font-mono text-xs uppercase
   - "SUPPLY CRISIS" badge: same pattern with crisis-supply colours
   - "SECURITY CRISIS" badge: same pattern with crisis-security colours

5. **Track Bar Shells section**: Two horizontal bars showing sample fill:
   - Severity track: Full-width bg-bg-surface container, inner fill at ~60% width bg-track-severity, height 6px, rounded-sm
   - Legitimacy track: Same pattern, inner fill at ~40% width bg-track-legitimacy
   - Labels in font-mono: "Severity: 3/5" and "Legitimacy: 0 / -2 to +2"

6. **Typography Samples section**:
   - "Syne Display" rendered in font-display font-bold text-xl
   - "DM Sans Body Text" rendered in font-body text-base
   - "IBM Plex Mono Data" rendered in font-mono text-sm uppercase tracking-wider

7. **Background & Border Samples section**: Four small panels showing bg-base, bg-panel, bg-surface, bg-elevated with labels, each with border-default borders.

8. **Resource Colours section**: Six small coloured dots in a row — PC, PO, Readiness, Stock, CRM, IC — with labels.

**Layout**: Use a single-column layout with max-w-4xl mx-auto px-8 py-12. Each section separated by border-t border-border-subtle pt-8 mt-8. Section titles in font-mono uppercase tracking-widest text-text-secondary text-xs mb-4.

**Critical rule**: ZERO hardcoded hex values. Every colour must come from a Tailwind utility class that resolves from the @theme tokens. This is the entire point of this component — proving the tokens work.

The component should be a simple functional component with no props, no state, no side effects. Pure render.
