```python
def _mock_client(handler) -> httpx.AsyncClient:
    return httpx.AsyncClient(transport=httpx.MockTransport(handler))
```

**Test 1 — Success (`test_health_check_success`):**
- Mock handler returns `httpx.Response(200, json={"choices": [{"message": {"role": "assistant", "content": "OK"}}]})`.
- Use `env_base` fixture for required env vars.
- Pattern:
  ```python
  with TestClient(app) as client:
      original = app.state.http_client
      app.state.http_client = _mock_client(handler)
      try:
          resp = client.get("/api/health/llm")
      finally:
          app.state.http_client = original
  ```
- Assert `resp.status_code == 200`
- Assert `body = resp.json()`:
  - `body["ok"] is True`
  - `isinstance(body["latencyMs"], int)` and `body["latencyMs"] >= 0`
  - `set(body.keys()) == {"ok", "latencyMs"}` (no extra keys on success)

**Test 2 — Auth failure (`test_health_check_auth_failure_401`):**
- Mock handler returns `httpx.Response(401, json={"error": "unauthorized"})`.
- Same swap pattern.
- Assert `resp.status_code == 200` (the endpoint always returns HTTP 200 — 09-CONTEXT.md).
- Assert body:
  - `body["ok"] is False`
  - `body["code"] == "auth_error"`
  - `body["status"] == 401`
  - `"LLM_API_KEY" in body["hint"]` (hint is actionable and mentions the env var)
  - `isinstance(body["latencyMs"], int)`

**Test 3 — Timeout (`test_health_check_timeout`):**
- Mock handler raises `httpx.TimeoutException("mock timeout", request=request)`. The `request=` kwarg is required by httpx's exception constructor (09-RESEARCH.md Q6).
- Same swap pattern.
- Assert `resp.status_code == 200`.
- Assert body:
  - `body["ok"] is False`
  - `body["code"] == "timeout"`
  - `body["status"] is None`
  - `body["hint"]` is a non-empty string containing "15 seconds" or "15s"
  - `isinstance(body["latencyMs"], int)`

**Test 4 — Auth-mode parity (`test_health_check_uses_configurable_auth_header`):**
- Set `LLM_AUTH_HEADER_NAME=api-key` and `LLM_AUTH_VALUE_PREFIX=` (empty string) via `monkeypatch.setenv`. Because `reset_settings_cache` is autouse, the Settings cache will be cleared on test entry. The `env_base` fixture sets `LLM_API_KEY`; verify the pattern used in `test_llm_auth_header.py` for how it overrides auth-mode vars specifically (it may call `get_settings.cache_clear()` explicitly after monkeypatch — copy whatever that file does).
- Mock handler captures `request.headers` and returns a valid 200 response.
- After the test call, assert:
  - The captured headers contain `api-key` with value equal to `LLM_API_KEY` env value (no "Bearer " prefix).
  - The captured headers do NOT contain an `Authorization` header (case-insensitive — use httpx's case-insensitive `headers` accessor or lowercase the keys).
- This test would fail if someone refactored `health.py` to copy `config_gen.py`'s hardcoded `Authorization: Bearer` pattern — that's the point.

**Test 5 — Extra headers parity (`test_health_check_forwards_extra_headers`):**

- Mock handler captures `request.headers` and returns a valid 200 response.
- Assert:
  - Captured headers contain `X-Corp-Trace: abc123`
  - Captured headers contain `X-Environment: test`
