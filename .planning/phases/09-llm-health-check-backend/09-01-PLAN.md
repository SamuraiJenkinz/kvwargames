**Imports (minimum):**

```python
import logging
import time

from fastapi import APIRouter, Request
from fastapi.responses import JSONResponse
from ..config import get_settings

logger = logging.getLogger(__name__)
router = APIRouter()
```

**Probe payload constant** (near top of file):
```python
_HEALTH_PROBE_MESSAGES = [{"role": "user", "content": "Reply with: OK"}]
_HEALTH_TIMEOUT_SECONDS = 15.0
```

**Handler `async def llm_health_check(request: Request) -> JSONResponse`:**

1. `settings = get_settings()` and `client = request.app.state.http_client` — reuse the shared `httpx.AsyncClient`; do NOT instantiate a new client. (See 09-RESEARCH.md Q4 and Pitfall 4.)

2. Build the auth header dict by copying the 3-line pattern from `backend/app/routers/llm.py` lines 79-84 verbatim — DO NOT copy from `config_gen.py` which hardcodes `Authorization: Bearer` and is wrong (09-RESEARCH.md Q1 and Pitfall 1):
   ```python
   auth_value = f"{settings.llm_auth_value_prefix}{settings.llm_api_key}".strip()
   headers = {
       settings.llm_auth_header_name: auth_value,
       "Content-Type": "application/json",
       **settings.get_extra_headers(),
   }
   ```

3. Build probe payload:
   ```python
   payload = {
       "model": settings.llm_model,
       "messages": _HEALTH_PROBE_MESSAGES,
       "max_tokens": 5,
       "temperature": 0,
   }
   ```

4. Record `t0 = time.monotonic()` then call:
   ```python
   response = await client.post(
       settings.llm_endpoint_url,
       json=payload,
       headers=headers,
       timeout=httpx.Timeout(_HEALTH_TIMEOUT_SECONDS),
   )
   ```
   The `timeout=` kwarg is a per-request override — this is critical for HEALTH-06 because the client-level default is `LLM_TIMEOUT_SECONDS` (60s by default). Without the override the 15s SLA is NOT enforced. (09-RESEARCH.md Pitfall 6.)

5. On the happy path: `response.raise_for_status()`, `data = response.json()`, extract `content = data["choices"][0]["message"]["content"]`. If `content` is falsy (empty string / None), raise a `ValueError` so the generic handler maps it to `invalid_response`. Compute `latency_ms = int((time.monotonic() - t0) * 1000)`. Log at INFO with fields `code="ok"`, `status=200`, `latencyMs=latency_ms`. Return `JSONResponse(status_code=200, content={"ok": True, "latencyMs": latency_ms})`.

6. **Exception handlers in this exact order** (ordering is load-bearing — see 09-RESEARCH.md Q8 and Pitfall 2: `TimeoutException` is a subclass of `RequestError` and MUST come first):

   a. `except httpx.TimeoutException:` → code=`timeout`, status=`None`, hint=`"LLM did not respond within 15 seconds — check network or provider latency"`

   b. `except httpx.HTTPStatusError as exc:` → dispatch on `exc.response.status_code`:
      - `401` or `403` → code=`auth_error`, hint=`"Authentication failed — check LLM_API_KEY in .env"`
      - `404` → code=`not_found`, hint=`"LLM endpoint not found — check LLM_URL path in .env"`
      - `429` → code=`rate_limited`, hint=`"Rate limited by LLM provider — retry in a moment"`
      - `>= 500` → code=`upstream_error`, hint=`f"LLM provider error (HTTP {status}) — try again or check provider status"`
      - Any other (e.g. 400) → code=`upstream_error` with the same hint formatted with the actual status
      - `status` field in the response body is always `exc.response.status_code` (an int)

   c. `except httpx.ConnectError as exc:` → discriminate TLS vs network by inspecting `exc.__cause__`:
      ```python
      import ssl
      if isinstance(exc.__cause__, ssl.SSLError):
          code, hint = "tls_error", "TLS handshake failed — check LLM_EXTRA_HEADERS and corporate proxy settings"
      else:
          code, hint = "network_error", "Cannot reach LLM endpoint — check LLM_URL in .env and network connectivity"
      ```
      status=`None`. **Note:** `__cause__`-based TLS discrimination is not a documented-stable httpx API per 09-RESEARCH.md Q8; if this proves flaky during testing, collapse both into `network_error` and note it in the SUMMARY.

   d. `except httpx.RequestError:` → code=`network_error`, status=`None`, same hint as the non-TLS ConnectError branch. (Catch-all for other request errors like `ReadError`, `WriteError`, `PoolTimeout`-that-isn't-TimeoutException-somehow, etc.)

   e. `except Exception:` → code=`invalid_response`. If the upstream returned 200 but we failed to parse (`KeyError`, `IndexError`, `JSONDecodeError`, the `ValueError` raised in step 5), `status=200`; otherwise `status=None`. Use a simple `status = 200 if "response" in locals() and response.status_code == 200 else None` or just always `status=200` since this branch is only reachable after a successful `raise_for_status()`. Hint=`"LLM returned an unexpected response shape — check LLM_URL points to a chat-completions endpoint"`.

7. In **every** failure branch: compute `latency_ms = int((time.monotonic() - t0) * 1000)`, log one INFO line with `{code, status, latencyMs}`, and return `JSONResponse(status_code=200, content={"ok": False, "code": code, "status": status, "hint": hint, "latencyMs": latency_ms})`. **HTTP status is ALWAYS 200** (09-CONTEXT.md response semantics + Pitfall 5) — the endpoint is healthy even when its dependency isn't.

**Response shape contract (stable — Phase 10 depends on this):**
- Success: `{"ok": true, "latencyMs": <int>}` — exactly two keys
- Failure: `{"ok": false, "code": <enum>, "status": <int|null>, "hint": <string>, "latencyMs": <int>}` — exactly five keys
- `code` enum values: `timeout | auth_error | not_found | rate_limited | upstream_error | network_error | tls_error | invalid_response` (stable strings)

**Logging format:** Use `logger.info("llm_health_check", extra={"code": code, "status": status, "latencyMs": latency_ms})` OR a simple f-string like `logger.info("llm_health_check code=%s status=%s latencyMs=%d", code, status, latency_ms)` — whichever matches the rest of the codebase. Do NOT log the probe payload, the upstream response body, or the API key. (09-CONTEXT.md logging decisions.)

**Do NOT define Pydantic response models.** The existing routers return plain dicts via `JSONResponse` (09-RESEARCH.md Q7). Stay consistent.

**Do NOT add caching, rate limiting, or debouncing** — every call fires a real probe (09-CONTEXT.md caching decisions).
