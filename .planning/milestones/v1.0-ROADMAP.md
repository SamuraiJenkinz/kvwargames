# Milestone v1.0: War Game Engine MVP

**Status:** ✅ SHIPPED 2026-04-15
**Phases:** 1–8 (8 phases)
**Total Plans:** 39

## Overview

Three-persona AI facilitation console for live policy tabletop exercises. The dependency chain was strict: TypeScript contracts and Zustand store stabilised before any other layer was built. The backend shipped as a thin credential proxy; all domain complexity lives in the React frontend. The LLM integration phase (Phase 6) received the most implementation time — context windowing, defensive JSON parsing, and state clamping correctness were architectural decisions made upfront, not retrofitted. Config generation from a text brief was deliberately deferred to Phase 7 because its prompt-engineering risk was independent of the core game loop.

## Milestone Summary

**Decimal Phases:** None (clean 8-phase execution).

**Key Decisions:**

- FastAPI + React/Vite over Next.js monolith — user prefers Python backend; rich UI needs React anyway (✓ Good: proxy stayed thin, frontend owned domain complexity)
- TypeScript interfaces before everything else — cascade rework cost of changing `GameState` post-Phase 1 would be high; lock-in was intentional (✓ Good: zero rework needed through Phase 8)
- Backend built Phase 2, before UI (Phases 3–5) — backend tests independently with curl; no stub-then-real round trip (✓ Good)
- UI layout against mock data (Phase 5) before LLM wiring (Phase 6) — decouples visual development from LLM iteration (✓ Good: entire visual layer shipped before any LLM cost)
- Config generation deferred to Phase 7 — prompt-engineering risk isolated from core game loop (✓ Good)
- LLM auth header configurable via `LLM_AUTH_HEADER_NAME` + `LLM_AUTH_VALUE_PREFIX` env vars — default Bearer preserves OpenAI byte-identical; Azure flip is env-only (✓ Good)
- `HISTORY_WINDOW_N=2` chosen empirically after 06-08 token budget measurement — 5124 prompt tokens on EDIP config forced tighter window than estimate (✓ Good: empirical over assumption)
- `structuredClone` over lodash.cloneDeep for pure state updater — Node 17+/modern browsers; zero dependency cost (✓ Good)
- Four-layer defensive parser (BOM/fence strip → JSON.parse → manual type guard → sort+dedupe) with zero `throw` statements, all failures return discriminated unions (✓ Good: no try/catch required at call sites)
- No server-side JSON parsing of LLM config output — frontend owns validation so errors reach facilitator (✓ Good)

**Issues Resolved:**

- Context overflow risk on 5-round scenarios — fixed via empirical `HISTORY_WINDOW_N=2` in 06-08
- Debrief export double-rendering post-debrief messages in Round-N transcripts — fixed in 08-05 (halt loop at lastDebriefIdx)
- Zustand v5 selector returning unstable `[]` reference caused infinite rerender — caught in 05-05, pattern documented
- Tailwind v4 purges template-literal class names — pre-baked lookup tables (PERSONA_META.bubbleClass, CAT_CHIP_CLASS) resolve
- Credential leakage risk — HAR evidence in 08-03 confirms zero browser-side auth headers

**Issues Deferred (Phase 9 backlog):**

- DEV-seed bug at `gameStore.ts:304` (setState-during-render React warning) — fix spec written into 08-02-LIVE-RUN.md
- R1 input first-character strip cosmetic in debrief export ("ound 1 is now live...")
- `crisisState` never auto-advanced despite `crisisSeverity` reaching 4 — prompt-engineering review needed

**Technical Debt Incurred:**

- Backend swallows upstream error detail (cosmetic; status code preserved)
- `uvicorn --reload` does not watch `.env` (developer docs only)
- Vite dev proxy returns 502+HTML when backend down — maps to INTERNAL_ERROR not LLM_UNREACHABLE (dev-only; production routes differently)
- Stale localStorage from prior-tenant apps (cosmetic; not security-relevant)
- Error-handling chain in `config_gen.py` kept as explicit duplication of `llm.py` — refactor to shared helper deferred
