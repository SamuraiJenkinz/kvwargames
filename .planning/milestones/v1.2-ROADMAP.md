# Milestone v1.2: Debrief Podcast

**Status:** ✅ SHIPPED 2026-04-19
**Phases:** 13–16 (4 phases)
**Total Plans:** 11

## Overview

Convert the end-of-session debrief into a three-voice MP3 podcast — Kent Valentina, Dr. Alistair Finch, and Dr. Michael Chen each read their own existing `isDebrief: true` messages through distinct ElevenLabs stock voices, stitched into a single session MP3. Facilitators get an inline player and a Download MP3 button adjacent to the existing markdown download. Zero browser-side credentials; the markdown export path remains the single source of truth for session memory and never shares a failure boundary with the audio path. Shape: mock-first to live-last, mirroring the v1.1 Tier-B precedent — Phase 13 locked the corporate-firewall proof and the 100%-mockable backend foundation (TTSProvider ABC, FakeTTSProvider, wargame-vocabulary text preprocessor); Phase 14 shipped the full end-to-end podcast flow against the fake backend (SSE endpoint, inline player, cache, cancel, skip-to-persona, Re-generate); Phase 15 added TTS health-check parity and empirically proved graceful degradation by flipping a garbage key mid-session and confirming the markdown debrief remained fully functional; Phase 16 completed the Tier-B live replay against a Scenario-2 fixture with real ElevenLabs v0.10 voice IDs on the target deployment host, producing a 782,966-byte stitched MP3 (52s, three distinct voices) with EDIP letter-by-letter pronunciation confirmed by operator listen-through.

**Plans**: 3 plans

Plans:
- [x] 14-01: Backend SSE sidecar endpoint (`POST /api/debrief/podcast` text/event-stream) + paired audio-by-token endpoint + `audio_generator.py` orchestrator (raw-bytes stitching with committed 700ms silence pad fixture, CBR per-segment offsets, in-process cache + token store, client-disconnect abort between personas) — completed 2026-04-18

- [x] 14-03: Visible UI: `PodcastSection` orchestrator + `GenerationPanel` + `PodcastPlayer` + `TranscriptPanel` + two `ConfirmDialog`-based modals + `ActionToolbar` three-state button row + human-verify checkpoint on dev server — completed 2026-04-18

**Details:**
Plan 14-01 shipped `POST /api/debrief/podcast` as an SSE endpoint using `sse_starlette.EventSourceResponse` with `raw_data=` to avoid double-JSON-encoding on the wire, paired with `GET /api/debrief/podcast/audio?token=...` for audio retrieval. `audio_generator.py` orchestrates synthesis as an async generator yielding plain dicts (`persona_done`, `done`, `error`) — keeps the generator HTTP-framework-agnostic and unit-testable. Raw-bytes stitching via `kent + SILENCE_BYTES + finch + SILENCE_BYTES + chen` consumes a committed 11,702-byte `silence_700ms.mp3` asset; no `pydub`/`ffmpeg`/vendored binaries because all ElevenLabs segments share `mp3_44100_128` CBR and MP3 frames are self-contained. Per-segment offsets computed from segment byte lengths with the same CBR formula used live and fake. In-process cache keyed by SHA-256 of `(kent_text + finch_text + chen_text + voice_ids)` enables PODGEN-07 cache-hit fast path (all three `persona_done` events emitted then `done` with `cached: true`, zero provider calls); cache-bust via `force_fresh=True` calls `cache.invalidate(key)` before synthesis. Client-disconnect abort-between-personas via `await request.is_disconnected()` polled after each segment. Plan 14-02 shipped `src/lib/podcastClient.ts` using `fetch + ReadableStream` (not `EventSource` — lacks POST body support needed to send persona texts/voice IDs), a standalone `usePodcastStore` (Zustand) kept distinct from the 700-line `gameStore` to avoid lifecycle pollution, FSM states `idle/generating/done/error`, blob URL lifecycle with revoke-before-create invariant (`URL.revokeObjectURL(prev)` at top of `startGeneration` and in `reset()`), `src/lib/mp3Filename.ts` using local-time `getHours/getMinutes` (NOT `toISOString`) so AEST facilitators get correct timestamps with `'session'` fallback, and `src/lib/wordCountEstimate.ts` with `WORD_COUNT_SOFT_CEILING=2000` plus 150-wpm + 3×delay estimation math. Plan 14-03 shipped seven React components — `ConfirmDialog.tsx` shared modal primitive (plain div with `role=dialog` + `aria-modal=true`, no `<dialog>` element because jsdom's `showModal()` is patchy), `WordCountConfirmDialog.tsx` and `RegenerateConfirmDialog.tsx` thin wrappers, `GenerationPanel.tsx` with per-persona `PersonaStatusRow`s (waiting/rendering/done) + discrete `count/3 × 100` progress bar + Cancel button + error banner, `PodcastPlayer.tsx` with `<audio controls>` (no `autoplay` attribute; imperative `audioRef.current.src = blobUrl; audio.load()` to avoid reconciler racing the blob URL), three Skip buttons seeking via backend-returned offsets, `timeupdate` listener walking offsets[] backwards for active-persona detection, "Now playing: {name}" label from `PERSONA_META[activePersona]`, `TranscriptPanel.tsx` collapsed-by-default with `generateDebriefMarkdown()` rendered in `<pre>` (zero new markdown dependency), `PodcastSection.tsx` status-gated orchestrator (idle→null, generating/error→GenerationPanel, done+blobUrl→PodcastPlayer), and `ActionToolbar.tsx` extended with a three-state podcast button row (idle→Generate, done→Download MP3 + Re-generate). The orchestrator mid-phase discovered and fixed a critical bug in `extractPersonaTexts` — original implementation filtered on `m.isDebrief` but `gameStore.buildPersonaMessage` never sets that flag; reanchored on `findLastIndex(m => m.type === 'debrief_divider')` then collected persona messages after it (commit `2c79de0`). Human-verify checkpoint on dev server with `TTS_PROVIDER=fake` + `FAKE_TTS_DELAY_SECONDS=0.5` confirmed SC1..SC6 end-to-end (Chrome + VLC); 608 frontend + 125 backend tests green at phase exit.

## Milestone Summary

**Decimal Phases:**

None. All four phases (13–16) were planned up front as part of the v1.2 roadmap; no mid-milestone insertions were required.

**Key Decisions:**

- Mock-first, live-last execution order (Phases 13/14/15 against FakeTTSProvider, Phase 16 first live call). Outcome ✓ Good — every UX surface debugged before any ElevenLabs quota spent.
- PODDEP-01 firewall spike as entry-gate task of Phase 13, not deferred to Phase 16. Outcome ✓ Good — confirmed `api.elevenlabs.io` reachability via operational precedent + HTTP 200 preflight on 2026-04-17 before any production provider code landed.
- Blocking streaming `audio/mpeg` delivery via SSE, not 202-then-poll or async job state. Outcome ✓ Good — backend stays stateless; per-persona progress UI is optimistic client-side animation, not a real progress feed.
- Raw-bytes MP3 concat with committed 700ms silence pad fixture — no `pydub` / `ffmpeg` / vendored binaries. Outcome ✓ Good — works because all ElevenLabs segments share `mp3_44100_128` CBR and MP3 frames are self-contained; cosmetic WMP duration-display quirk accepted as expected consequence.
- Parallel `/api/health/tts` endpoint, not extended `/api/health/llm`. Outcome ✓ Good — LLM-down hard-fails Launch, TTS-down is a soft warning; one `body.ok` cannot carry both signals; PODRES-02 invariant (Launch-never-blocked) preserved by construction.
- Graceful degradation empirically verified by flipping the key to garbage, not just mocked. Outcome ✓ Good — SHA-256 `b00eda86…` of downloaded debrief.md pins the invariant that the markdown path survives ElevenLabs failure.
- Two-endpoint error-code divergence accepted as expected behaviour (`/v1/user` → `auth_error`, `/v1/text-to-speech/{voice_id}` → `upstream_error`). Outcome ✓ Good — strengthened the verification by proving the code-dispatch table handles multiple concurrent error codes correctly.
- `ServerSentEvent raw_data=` not `data=` — `data=` double-JSON-encodes on the wire. Outcome ✓ Good — SSE consumer received correct payloads.
- `fetch + ReadableStream` not `EventSource` in the frontend SSE consumer — `EventSource` lacks POST body support. Outcome ✓ Good — enabled sending persona texts/voice IDs in the request body.
- Separate `usePodcastStore` not a `gameStore` slice — podcast lifecycle is distinct from game state. Outcome ✓ Good — avoided polluting the 700-line `gameStore`.
- `extractPersonaTexts` anchored on last `debrief_divider` message, not `isDebrief` flag. Outcome ✓ Good — mid-phase orchestrator correction (commit `2c79de0`) unblocked the human-verify checkpoint; the root cause was a design mismatch between the extraction assumption and `gameStore.buildPersonaMessage` tagging.
- No `autoplay` on the `<audio>` element (PODPLAY-01). Outcome ✓ Good — explicit absence regression-tested; browser policy would suppress autoplay anyway.
- Blob URL not revoked on Download MP3 click. Outcome ✓ Good — store owns lifecycle; premature revoke breaks replay.
- No `<dialog>` element for confirm dialogs — plain div with `role=dialog`. Outcome ✓ Good — avoided jsdom `showModal()` gap and focus-trap complexity.
- `_make_http_client()` factory for testability in `/api/health/tts`. Outcome ✓ Good — global `httpx.AsyncClient` patching caused `RecursionError` and broke lifespan `aclose()`; narrow factory is the correct patch target.
- `?force=true` writes back to cache after bypass (bypasses read only, not write). Outcome ✓ Good — prevents stale cache after manual Re-check.
- One `vi.mock` scope per test file (mid-gen failure tests in a separate file). Outcome ✓ Good — avoided 8 test failures from mock hoisting that would replace real `generatePodcast` across unrelated tests.
- `/api/config/tts-voices` as a dedicated endpoint, not a generic config field. Outcome ✓ Good — keeps fake-vs-elevenlabs voice-ID dispatch isolated; replaced hardcoded sentinel strings in `ActionToolbar.tsx`.
- No Tier-A preprocessor fixes needed after live listen-through. Outcome ✓ Good — EDIP letter-by-letter confirmed on first pass across all 3 segments; Phase 13 ACRONYMS dict entry was sufficient.
- No Tier-B deferrals seeded into v1.3 VOICE-01 backlog. Outcome ✓ Good — all three stock voices (Sarah/George/Eric) delivered distinct intelligible audio on first pass.

**Technical Debt Incurred:**

- **TD-v1.2-01 (Low):** WMP reports incorrect duration (~11–13s) for the stitched MP3 because the raw-bytes concat preserves the first segment's Xing/VBRI header which encodes only that segment's frame count. Chrome, VLC, and the browser `<audio>` element all scan all MPEG frames and report correct playback duration. Full audio plays end-to-end without defect. Observed in both the fake-provider run (14-VERIFICATION.md) and the live ElevenLabs run (16-LIVE-VERIFICATION.md §4 ~3s CBR variance). Expected consequence of the no-pydub stitching architectural decision. Input captured as candidate for v1.3 if `pydub`/`ffmpeg` is ever introduced. No action this milestone — display-only cosmetic.
