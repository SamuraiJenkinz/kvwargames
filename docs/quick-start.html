KV WarGame Quick Start

For the facilitator

Your first five minutes.

This is all you need to open the tool, launch a scenario, and put a question to the AI table. Read it once before your session. Keep the longer user guide within reach for everything else.

Before you begin

Prerequisites

Your IT team will have given you a URL — something like http://wargame.internal:8000. You will run everything in a browser on the facilitation laptop. There is no login, no install, no app store. Chrome, Edge, or Firefox are all fine.

The tool does not replace you. It adds three AI voices — Kent Valentina, Alistair Finch, and Michael Chen — that respond in-character to what you describe at the table. You run the room; the tool tracks state and keeps the fiction coherent.

The five steps

  1. Open the app and choose a path.

    Load the URL. You will see two cards: Load Config / EDIP Default and Generate from Brief. For your first session, pick the first one. The EDIP scenario is pre-loaded.

  2. Review the JSON and launch.

    The editor shows the scenario config. You do not need to edit it — just scroll through the summary on the right: two scenarios, four teams, eleven cards. Click Launch Scenario 1 to begin.

    Tip If you pasted your own JSON and the Launch buttons are grey, check the red error note beneath the editor — it names the line and column of the problem.
  3. Orient yourself to the game screen.

    Three columns. The left shows live state — crisis severity, legitimacy, and the four team resource cards. The centre is the chat feed where the personas will respond. The right is a reference panel with tabs for cards, national actions, and a facilitator guide. You will consult it often.

  4. Send your first event.

    Type a short, in-world description into the input bar at the bottom — for example, “Team A announces they are rushing an industrial stockpile to the border.” Press Enter. One to three personas will answer in character. The state panel on the left may animate as resources change.

    If something goes wrong A red bubble with a Retry button means the AI reply was malformed — not your fault. Click Retry. Your message, the personas, and the state are preserved.
  5. Advance rounds, and end when ready.

    When the table is ready to move forward, click Advance to Round 2. Finch delivers the round’s inject, then Kent frames the key tension. When the session is over, press End Game & Debrief — all three personas give closing remarks, then a Download Debrief button appears. It saves a Markdown file of the full transcript.

  6. Optional — generate a three-voice podcast.

    After the debrief lands, a Generate Podcast button appears next to Download Debrief. Click it. Each persona reads their own closing remarks in a distinct voice; Kent, Finch, then Chen. A player appears inline with Skip-to-persona buttons and a Download MP3 button alongside the existing Markdown download.

    If audio is unavailable A small TTS badge on the setup screen shows whether the podcast service is reachable. If it is amber (or if generation fails mid-run), the Markdown debrief still downloads normally — the two paths are independent by design.

“The tool listens and responds. The facilitator holds the room.”

What to do next

Run a practice session before your first live one. Ten minutes of Scenario 1 with imagined events will tell you more than any guide. When you are ready for the deeper dive — state clamping, card routing, generating a new scenario from a brief, the debrief format — read the full user guide.
